# Exercise 5 - Lombok

## Objective
Use Lombok annotations to reduce repeated Java code in a simple Employee class.

## Problem Statement
Create an Employee class and use Lombok annotations so the class can be used without writing common methods manually.

## Approach Used
I used @Getter, @Setter, @NoArgsConstructor, @AllArgsConstructor, and @ToString on the Employee class. Then I created a Main class to show the result.

## Project Structure
- Employee.java
- Main.java

## Execution Steps
1. Go to this folder.
2. Run the main class.
3. Check the output in the console.

## Sample Output
- Employee details are printed
- Getter and setter methods are available

## Learning Outcome
Lombok helps keep simple classes short and readable.

## Observations
- I noticed that Lombok reduces the need to write getters and setters manually.
- The class remains easy to read.
- This is useful when the model is simple and repetitive.

## Things I Learned
- Lombok saves time for common methods.
- It is helpful for small data classes.
- The generated methods are easy to understand once the annotations are known.
